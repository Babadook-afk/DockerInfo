# DockerInfo

Репозиторий для хранения структурированных заметок, документации и примеров, связанных с технологией Docker.

## О Docker

### Определение

**Docker** — это платформа с открытым исходным кодом для разработки, доставки и запуска приложений в изолированных средах, называемых **контейнерами**. Контейнеризация позволяет упаковать приложение со всеми его зависимостями (библиотеками, системными инструментами, кодом и настройками) в стандартизированный блок, который может быть запущен на любой системе, где установлен Docker.

Ключевые преимущества:
*   **Изоляция:** Приложения в контейнерах не влияют друг на друга и на хост-систему.
*   **Переносимость:** "Собран один раз — работает везде".
*   **Эффективность:** Контейнеры разделяют ядро ОС хоста, что делает их гораздо легче и быстрее традиционных виртуальных машин.
*   **Скорость развертывания:** Контейнеры создаются за секунды.

### История появления и развития

*   **2013:** Соломон Хайкс представляет Docker на конференции PyCon как проект с открытым исходным кодом. Изначально он был построен на базе технологий Linux: **cgroups** и **namespaces**, которые обеспечивают изоляцию процессов.
*   **2014:** Выпуск версии 1.0. Docker начинает стремительно набирать популярность, становясь стандартом де-факто для контейнеризации. Появляется экосистема: **Docker Hub** (реестр образов), **Docker Compose** (оркестрация многоконтейнерных приложений).
*   **2015-2017:** Формирование **Open Container Initiative (OCI)** при участии Docker, Linux Foundation и других ключевых игроков. Цель — стандартизация форматов контейнеров и сред выполнения (runc). Рост конкуренции и появление инструментов оркестрации (Kubernetes, Docker Swarm).
*   **2017-настоящее время:** Docker разделяет свою монолитную архитектуру на отдельные компоненты (Moby Project). Kubernetes становится доминирующим оркестратором. Docker Inc. фокусируется на разработческих рабочих процессах (Docker Desktop), а корпоративные решения продаются (Mirantis). Сегодня Docker — неотъемлемая часть CI/CD, DevOps и облачных нативных разработок.

### Современное состояние

Docker прочно укрепился в качестве фундаментального инструмента для разработчиков и DevOps-инженеров. Основные тенденции:
1.  **Docker Desktop:** Стандартный инструмент для локальной разработки под Windows, macOS и Linux.
2.  **Интеграция в CI/CD:** Контейнеры — основа пайплайнов сборки и тестирования в GitLab CI, GitHub Actions, Jenkins и др.
3.  **Облачные сервисы:** Все крупные облачные провайдеры (AWS, Azure, GCP) предлагают managed-сервисы для работы с контейнерами (ECS, AKS, GKE) и собственными реестрами образов (ECR, ACR, GCR).
4.  **Безопасность:** Повышенное внимание к сканированию образов на уязвимости (trivy, docker scout), использованию минимальных базовых образов (Alpine, Distroless) и подписыванию образов (Notary).

### Docker Hub и готовые образы

**Docker Hub** — крупнейший в мире облачный реестр (registry) для хранения и распространения образов Docker. Это "GitHub для контейнеров".

*   **Официальные образы (Official Images):** Курируемые Docker образы основных проектов (например, `nginx`, `postgres`, `python`, `node`). Проходят проверку на безопасность и актуальность.
*   **Проверенные издатели (Verified Publishers):** Образы от известных компаний (например, `microsoft/`, `mongodb/`).
*   **Общедоступные репозитории:** Любой пользователь может загружать свои образы (например, `username/my-app`).

**Пример использования готового образа:**
```bash
docker pull nginx:alpine # Скачать официальный образ Nginx на базе Alpine Linux
docker run -d -p 80:80 nginx:alpine # Запустить контейнер на порту 80

### **Часть 2: Пошаговая инструкция по выполнению задания (Git + VS Code)**

Следуйте этим шагам, чтобы создать репозиторий и разместить в нём файл.

1.  **Подготовка в VS Code:**
    *   Откройте VS Code.
    *   Создайте новую папку на вашем компьютере специально для этого задания (например, `DockerInfo`).
    *   В VS Code откройте эту папку через `File` -> `Open Folder...`.

2.  **Инициализация Git-репозитория:**
    *   Откройте встроенный терминал в VS Code (`Terminal` -> `New Terminal`).
    *   Убедитесь, что вы находитесь в папке проекта. Затем выполните команду:
        ```bash
        git init
        ```
    *   Это создаст скрытую папку `.git` для контроля версий.

3.  **Создание и заполнение файла `README.md`:**
    *   В левой панели VS Code нажмите на иконку создания нового файла.
    *   Назовите файл `README.md` (обязательно с таким расширением).
    *   **Скопируйте ВЕСЬ текст из "Части 1" этой инструкции** и вставьте его в созданный файл.
    *   Сохраните файл (`Ctrl+S`).

4.  **Создание репозитория на GitHub:**
    *   Залогиньтесь на [github.com](https://github.com) или [github.ru](https://github.ru).
    *   Нажмите кнопку `+` вверху справа и выберите `New repository`.
    *   В поле **Repository name** введите `DockerInfo`.
    *   Можно добавить описание: "Репозиторий для заметок о Docker".
    *   **Не** добавляйте `README.md`, `.gitignore` или `license` на этом этапе (мы уже создали файл локально).
    *   Нажмите `Create repository`.

5.  **Связывание локального репозитория с удаленным (GitHub):**
    *   После создания репозитория GitHub покажет вам команды для связи.
    *   В терминале VS Code выполните следующие две команды (замените `ваш-username` на ваш реальный логин на GitHub):
        ```bash
        git remote add origin https://github.com/ваш-username/DockerInfo.git
        git branch -M main
        ```

6.  **Фиксация изменений и отправка на GitHub:**
    *   В терминале VS Code выполните последовательно:
        ```bash
        git add README.md          # Добавляем наш файл в область staged
        git commit -m "Initial commit: Added comprehensive README about Docker"  # Фиксируем изменения
        git push -u origin main    # Отправляем изменения на GitHub
        ```
    *   Вас могут попросить ввести логин и пароль (токен) от GitHub. Используйте **Personal Access Token (Classic)** с правами `repo`.

7.  **Проверка и отправка ссылки:**
    *   Обновите страницу вашего репозитория на GitHub. Вы должны увидеть красивый, отформатированный файл `README.md`.
    *   Скопируйте адрес этой страницы из адресной строки браузера (например, `https://github.com/ваш-username/DockerInfo`).
    *   **Это и есть искомая ссылка.** Отправьте её преподавателю в чат Собрания или в Telegram.

**Готово!** Вы успешно создали репозиторий, заполнили его структурированной информацией о Docker с использованием Markdown и отправили его на GitHub, продемонстрировав владение базовым рабочим процессом Git.
